## 2️⃣ 配置文件 (configs/)

这些文件定义了机器人训练和环境的**所有参数**。


| 文件                     | 功能           | 作用说明                                                                                                                                   |
| ------------------------ | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `a1_custom_config.py`    | **V1配置**     | 初始版本的A1机器人配置，包含环境参数（1024并行环境）、地形参数（课程学习）、奖励函数、PPO训练参数（1500迭代）等。适合RTX 3050 Ti 4GB显存。 |
| `a1_custom_config_v2.py` | **V2优化配置** | 改进版本：2048并行环境、更大的速度范围、增强的奖励函数（步态奖励增加到2.0）、5000迭代、更低的熵系数和学习率，用于更深入的训练。            |

### 配置类说明

每个配置文件包含两个主要类：

- **`A1CustomCfg` / `A1CustomCfgV2`**: 环境配置类，继承自 `LeggedRobotCfg`

  - `env`: 环境参数（并行环境数、观测维度等）
  - `terrain`: 地形参数（类型、课程学习、地形比例）
  - `commands`: 指令参数（速度范围、采样间隔）
  - `rewards`: 奖励函数配置
  - `control`: 控制参数（PD增益、控制频率）
- **`A1CustomCfgPPO` / `A1CustomCfgPPOV2`**: PPO训练配置类

  - `policy`: 网络结构（Actor/Critic隐藏层维度）
  - `algorithm`: 算法参数（学习率、裁剪参数、熵系数）
  - `runner`: 运行参数（最大迭代数、保存间隔）

## 3️⃣ Legged Gym 框架 (legged_gym/)

这是ETH Zurich开发的**足式机器人训练框架**，本项目在此基础上进行开发。

### 核心环境模块


| 文件                                          | 功能               | 作用说明                                                                                                                                                      |
| --------------------------------------------- | ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `legged_gym/envs/base/legged_robot.py`        | **足式机器人基类** | 核心环境类，继承自`BaseTask`。实现 `step()`、`post_physics_step()`、`compute_reward()`、`compute_observations()` 等方法，处理物理仿真、奖励计算、观测生成等。 |
| `legged_gym/envs/base/legged_robot_config.py` | **基础配置类**     | 定义`LeggedRobotCfg` 和 `LeggedRobotCfgPPO`，是所有机器人配置的基类。                                                                                         |
| `legged_gym/envs/base/base_task.py`           | **任务基类**       | 定义强化学习任务的通用接口和基础功能。                                                                                                                        |
| `legged_gym/envs/base/base_config.py`         | **配置基类**       | 配置类的基类，提供配置解析功能。                                                                                                                              |
| `legged_gym/envs/a1/a1_config.py`             | **官方A1配置**     | Legged Gym自带的A1机器人配置示例。                                                                                                                            |

## 4️⃣ RSL-RL 算法库 (rsl_rl/)

ETH Zurich开发的**强化学习算法库**，实现了PPO等算法。

### 核心算法


| 文件                                       | 功能                 | 作用说明                                                                                                                  |
| ------------------------------------------ | -------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `rsl_rl/algorithms/ppo.py`                 | **PPO算法实现**      | 实现近端策略优化（Proximal Policy Optimization）算法，包含`init_storage()`、`update()` 等核心方法，支持自适应学习率调度。 |
| `rsl_rl/runners/on_policy_runner.py`       | **训练运行器**       | 管理训练循环，处理数据收集、策略更新、日志记录、模型保存/加载等功能。                                                     |
| `rsl_rl/modules/actor_critic.py`           | **Actor-Critic网络** | 定义策略网络和价值网络的架构，使用MLP实现。                                                                               |
| `rsl_rl/modules/actor_critic_recurrent.py` | **循环Actor-Critic** | 带LSTM的Actor-Critic网络，用于需要记忆的任务。                                                                            |
| `rsl_rl/storage/rollout_storage.py`        | **经验存储**         | 存储训练过程中的轨迹数据（观测、动作、奖励、价值等），支持GAE计算。                                                       |
| `rsl_rl/env/vec_env.py`                    | **向量化环境接口**   | 定义向量化环境的抽象接口。                                                                                                |

### 工具模块


| 文件                    | 功能         | 作用说明                                 |
| ----------------------- | ------------ | ---------------------------------------- |
| `rsl_rl/utils/utils.py` | **工具函数** | 提供神经网络初始化、观察值归一化等工具。 |

### 核心工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                       训练流程                               │
├─────────────────────────────────────────────────────────────┤
│  1. train_a1.py / train_a1_v2.py                            │
│     ├── 加载 a1_custom_config.py / a1_custom_config_v2.py   │
│     ├── 调用 task_registry.register() 注册任务              │
│     ├── 创建 LeggedRobot 环境                               │
│     ├── 创建 OnPolicyRunner (PPO)                           │
│     └── 开始训练循环                                        │
│                                                             │
│  2. LeggedRobot (legged_robot.py)                           │
│     ├── step() → 执行动作 + 物理仿真                        │
│     ├── post_physics_step() → 计算奖励和观测                │
│     └── compute_reward() → 根据配置计算各种奖励             │
│                                                             │
│  3. OnPolicyRunner (rsl_rl)                                 │
│     ├── learn() → 主训练循环                                │
│     ├── 收集轨迹数据 (RolloutStorage)                       │
│     ├── PPO.update() → 策略更新                             │
│     └── 保存模型和TensorBoard日志                           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                       演示流程                               │
├─────────────────────────────────────────────────────────────┤
│  1. play_demo_*.py                                          │
│     ├── 注册任务并创建环境                                  │
│     ├── 加载训练好的模型 (.pt文件)                          │
│     └── 循环推理并渲染                                      │
│                                                             │
│  2. record_*.py                                             │
│     ├── 无头模式运行                                        │
│     ├── 捕获每一帧图像                                      │
│     └── 生成GIF或视频                                       │
└─────────────────────────────────────────────────────────────┘
```
